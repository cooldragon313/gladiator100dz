## § 1 戰前層

### 1.1 派遣制（取代無限日刷）

**現況問題**：
- 按鈕點日日刷、一天 4 次、沒有敘事合理性
- 主人為何讓奴隸隨便送死？不合理

**新機制**：
- 每日 **25% 機率**觸發派遣事件（平均 4 天一次，範圍 3-5 天）
- 首次觸發需：**fame ≥ 3**（前 10 天累積沙洗/訓練觀察）
- 按鈕**永久移除**——全部走事件驅動

### 1.2 派遣事件流程

```
【派遣事件】
侍從走進訓練場。

侍從：今日競技有一場。對手——[對手名字]。
侍從：是個[流派類型]型選手。[一句特徵描述]
侍從：[若達傷勢情報門檻] 聽說他前幾天[受傷部位]還帶點傷。

（侍從停頓。）
侍從：主人問——有誰自願？

[若玩家達標 fame + 主人好感 → 顯示 2 選項]
  ▸ 我去。（自願，主人好感 +3）
  ▸ 沉默。（棄權 → 進入隨機抽選，50% 被點到）

[若未達標 → 只顯示]
  ▸ （你沒資格站出來。侍從的目光掃過——停在......）
  → 50% 機率你被點
  → 50% 機率別人被抽走（這天跳過）
```

### 1.3 自願門檻（階梯式）

| 對手等級 | Fame 門檻 | 主人好感門檻 |
|---|---|---|
| 初等（Day 10-20）| 3 | 10 |
| 中等（Day 20-40）| 8 | 20 |
| 上等（Day 40-60）| 20 | 40 |
| 精英（Day 60-80）| 40 | 60 |

**規則**：達標才能**自願**；不達標只能**被動抽**（無法拒絕）。

### 1.4 對手資訊揭露

派遣事件**必揭露**：
- ✅ 名字
- ✅ 流派類型（力量/敏捷/技術/堅韌/意志）
- ✅ 裝備（武器 + 護甲）
- ⚠️ **傷勢**（需情報門檻 or 付費）

玩家據此選擇：
- 自己武器對不對（鈍器 vs 板甲 / 輕武器 vs 速型）
- 要不要去找赫克托買情報 / 下毒
- 要不要自願（如果不利）

### 1.5 對手池（每 tier 5 人隨機抽）

每 tier 定義 **5 個預設對手**（不完全隨機生成，增加辨識度）：

```js
OPPONENT_POOL[tier] = [
  {
    id: 'bear_heros',
    name: '熊羆赫魯斯',
    style: 'STR',          // 力量型
    title: '鈍器壓制',
    weapon: 'warHammer',
    armor: 'ironPlate',
    shield: 'ironShield',
    lore: '聽說他用重錘砸碎過新人頭骨。',
    statMod: { STR:+5, AGI:-3 },
  },
  // ...4 more
]
```

每場派遣從 tier 對應池子**隨機抽 1**；對戰到可以重複（大人物會再約戰）。

### 1.6 赫克托黑市（準備層主角）

接現有 `hector_deal_intro`（npc.js:605）擴充服務清單：

| 好感 | 服務 | 價格（依 tier 1/2/3/4）| 效果 | 成功率 |
|---|---|---|---|---|
| 10 | 📋 對手詳情 | 3 / 5 / 8 / 12 金 | 揭露對手屬性、裝備評價、傾向招式 | 100% |
| 30 | 🔍 對手傷勢 | 5 / 10 / 15 / 25 金 | 揭露對手當前所有傷勢 + 位置 | 100% |
| 50 | ☠ 戰前下毒 | 10 / 20 / 35 / 50 金 | 對手 HP -15%，戰鬥中偶發嘔吐（失回合）| 80% |
| 70 | 😈 散播謠言 | 15 / 30 / 50 / 80 金 | 對手 WIL -5、心情 -20，壓制易 | 85% |
| 90 | ⚰ 買通陪練 | 30 / 60 / 100 / 150 金 | 對手戰前受傷（部位擲骰，中傷）| 90% |

**服務限制**：
- 每次派遣**最多 2 個服務**（例如：情報 + 下毒，不能三管齊下）
- 同服務同一派遣只能買一次

**失敗後果**（成功率沒中）：
- 下毒失敗：錢沒退 + 赫克托好感 -3（他吃力不討好）
- 謠言失敗：錢沒退 + 對手 WIL +5（反效果，謠言激勵對手）
- 陪練失敗：錢沒退 + 主人好感 -3（主人懷疑）

**道德軸觸發**：
- 下毒 / 謠言 / 買通 → `moral:mercy:negative (weight:1)`
- 單純情報 → 無道德影響

## § 2 戰中層（現有引擎，動最少）

### 2.1 不動的部分

- ATB 節奏、攻擊/格擋/閃避判定
- 路線系統（憤怒/技巧/穩守）
- 屬性換算公式
- 名聲壓制 / 意志抗壓
- 武器/護甲的 `eqBonus` 套用

### 2.2 加的 2 條粗規則

只加**兩條**克制規則，不做 5×3 表：

| 規則 | 效果 |
|---|---|
| 🔨 **鈍器 vs 板甲** | 玩家用 hammer/warHammer vs 對手 ironPlate → 傷害 ×1.15 |
| 🗡 **刺擊 vs 板甲** | 玩家用 dagger/spear vs 對手 ironPlate → 傷害 ×0.85 |

**實作**：`battle.js` 傷害計算鏈加一個 `_weaponArmorMult(attackerWeapon, defenderArmor)` 函數，只判這兩種情形，其他回 1.0。

### 2.3 中毒 debuff（赫克托服務連動）

若派遣前對手被下毒，`Battle.startFromConfig` 讀入旗標：
- 對手起始 HP 設定為 `opp.hpMax × 0.85`
- 戰鬥中每 8-12 秒（隨機）對手跳一次「嘔吐」log，損失 1 回合攻擊
- 視覺：對手 slot 臉色加綠色濾鏡（CSS class `bt-slot-poisoned`）

## § 3 戰後層（輕量）

### 3.1 受傷擲骰（加大現有）

**現有**：只有低體力訓練會受傷。
**新增**：戰鬥結束時必擲傷勢骰。

| 情境 | 擲骰 |
|---|---|
| S 級碾壓（勝）| 0%（毫髮無傷）|
| A/B 勝 | **25% 機率擲輕傷** |
| C 慘勝 | **50% 擲輕-中傷** |
| A 失敗（存活）| **70% 擲中傷** |
| B 失敗（存活）| **100% 擲中傷，30% 重傷** |
| C 慘敗 | 死亡判定（S 級死亡結局）|

### 3.2 部位由武器類型決定（3 條規則）

不做 5×4 表，改**武器類別輪替選部位**：

| 武器類別 | 主要部位 | 次要 |
|---|---|---|
| 🔨 **鈍器**（hammer/warHammer）| 頭 | 軀幹 |
| 🗡 **刺擊**（dagger/spear）| 軀幹 | 腿 |
| ⚔ **劈砍**（shortSword/longSword/heavyAxe）| 軀幹 | 手 |

**擲法**：70% 主要部位 / 30% 次要部位。

**雙向適用**：
- 玩家被對手的武器打 → 用**對手武器類別**查表
- 對手被玩家武器打 → 用**玩家武器類別**查表

### 3.3 對手傷勢持久化

**關鍵機制**：對手受的傷會**持續存在**。

- 對手池裡每個對手有 `wounds = []` 欄位
- 戰鬥結束後對手傷勢寫回池子
- 下次該對手再派遣 → 顯示「[對手名字]（腿還帶著上次你給的傷）」
- 對手屬性按傷勢 debuff（沿用 wounds.js 的懲罰表）

**清理**：每 20 天重置對手池（對手休養好了），避免累積無限。

### 3.4 名聲與金錢

**勝利獎勵**：

| Tier | Fame | 金錢 |
|---|---|---|
| 初等 | 3-8 | **2-5 金** |
| 中等 | 6-14 | **4-10 金** |
| 上等 | 10-20 | **8-20 金** |
| 精英 | 15-30 | **15-40 金** |

- S 級 **名聲 + 金錢 ×1.5**
- A 級 照表
- B 級 ×0.75
- C 級 ×0.5

**敗北懲罰**：
- A 敗 fame -5
- B 敗 fame -3
- 失敗不賠錢（已經很慘了）

100 天估計獎勵：**打贏 15-20 場 → 累計 150-300 金 + fame 150+**。

## § 5 百日祭壓力

### Day 100 Champion 存活門檻（暗線）

- Fame ≥ 50
- 主武器 tier ≥ 3
- 護甲 tier ≥ 2
- 頭盔流派 tier ≥ 3
- 任一屬性 ≥ 30

**任一不達標 → Champion 碾壓**（battle.js 讀這些檢查，強化敵人屬性）。

玩家在 100 天內必須：
- 打贏派遣（約 15-20 次機會）
- 累積 fame 50+
- 跟葛拉混好升級主武器
- 專一流派刷到 tier 3+

→ **時間壓力明確**，選擇有代價（專精 vs 全面）。

## § 10 成功案例對照（Outward）

Outward 遊戲魅力拆解：
- 戰鬥本身：**很簡單**（三擊連段、簡單招式）
- 準備層：**極深**（煉藥、設陷阱、研究怪物、背包管理、體力管理）
- 玩家打贏 boss 不是因為 DPS 高，是因為**準備充分**

本專案對應：
- 戰鬥：**現有 ATB 引擎**（已深）
- 準備：**赫克托黑市 + 派遣揭露 + 裝備選擇**（要加）
- 玩家打贏 tough 對手不靠刷屬性，靠**買情報+選武器+下毒**

這種設計讓**角色養成有意義**（戰力有基礎）+ **每場戰鬥有思考**（準備決定勝敗），雙贏。
