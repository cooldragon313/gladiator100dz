# 多屬性判定任務模式

> 本模式適用於夜間事件與未來的挑戰型任務。
> 最後更新：2026-04-18

---

## 這是什麼

一個任務**連續考驗 2-4 項玩家屬性**，每項分開判定。失敗任何一項 = 該任務失敗，但給小獎慰問；全通過 = 完整獎勵 + 後續選擇。

典型長度：**3 階段**（WIL → AGI → DEX 這樣的排序）

---

## 為什麼用這種模式

### 對比：單一判定
「你有沒有 20 以上 STR？有的話給你獎勵，沒有就滾」
→ 玩家只會練 STR。其他屬性無意義。

### 多階段判定
「耐心等 + 速度追 + 手巧抓」
→ 玩家要均衡訓練。單點爆衝沒用。
→ 每個失敗點都有專屬對白 → 重玩也有戲

---

## 結構圖

```
【白天】NPC 接近 → ChoiceModal「接不接」
          ↓ 接 → 設 night_pending_<questId>
          ↓ 不接 → 結束，某些任務可能重試
【夜間（Slot 7）】
    開場 DialogueModal（場景建立）
       ↓ [特性對白支線 — prideful/cruel/kindness/neurotic 等]
    階段 1 判定（如 WIL）
       ├── 失敗：對白（暗示 WIL 不夠）+ 慰問獎（5% 獎勵）+ END
       └── 過：中段對白（可再有特性變化）
    階段 2 判定（如 AGI）
       ├── 失敗：對白（暗示 AGI 不夠）+ 慰問獎（30% 獎勵）+ END
       └── 過：接近完成
    階段 3 判定（如 DEX）
       ├── 失敗：對白（暗示 DEX 不夠）+ 慰問獎（60% 獎勵）+ END
       └── 過：完整獎勵
    後續 ChoiceModal（放生 / 殺 / 獻給 NPC 等）
       ↓ 每個選項不同效果
【結束】任務 flag 鎖定（本次 run 不再重試）
```

---

## 階段門檻範例

### 範本 A：新手可及（Day 3-5 接得了）
```
階段 1: WIL ≥ 12
階段 2: AGI ≥ 15
階段 3: DEX ≥ 15
```
Day 1 預設 10/10/10，練 3-4 天即可。

### 範本 B：中期挑戰（Day 10-20）
```
階段 1: WIL ≥ 18
階段 2: AGI ≥ 20
階段 3: DEX ≥ 20
```
需要專心練某幾條屬性才行。

### 範本 C：高難度（Day 30+）
```
階段 1: WIL ≥ 25
階段 2: AGI ≥ 25
階段 3: DEX ≥ 25
階段 4: STR ≥ 20（額外：收尾）
```
逃獄類型。全屬性均衡達成才有機會。

---

## 失敗慰問獎的分層

為了讓玩家**失敗也有感**：

| 失敗在哪 | 獎勵 |
|---|---|
| 階段 1 | 5%（幾乎沒有，主要是敘事） |
| 階段 2 | 30%（一半食物 + 好感 +3） |
| 階段 3 | 60%（食物 + 好感 +5 + 部分 flag） |
| 全通過 | 100%（完整獎勵） |

**原則**：走得越遠，收穫越多。但全通過才有「特別」的東西（寵物 / 道具 / storyReveal）。

---

## 對白的特性變化規範

每個任務**至少**要為以下特性寫專屬對白：

| 特性 | 性格 | 典型對白口氣 |
|---|---|---|
| （無 / 基礎） | 中立 | 標準敘述 |
| prideful | 自負 | 「一隻老鼠？別侮辱我」 |
| cruel | 殘暴 | 「抓到拿來加菜」 |
| kindness | 溫柔 | 「希望不用傷到牠」 |
| neurotic | 神經質 | 「要是被咬了怎麼辦？」 |
| coward | 膽小 | 「老鼠會咬人吧？」 |
| iron_will | 鐵意志 | 無多餘念頭，專注 |
| merciful | 寬厚 | 「能放就放」 |

並非每個任務都要全部寫完，但**基礎 + 至少 3 種特性變化**是最低標準。

## 判定的實作

### 程式碼端

```js
function _stage1_wait(p) {
  const wilThreshold = 15;          // 寫在程式碼，玩家看不到
  if (p.WIL >= wilThreshold) {
    // 成功對白
    DialogueModal.play(SUCCESS_LINES, { onComplete: () => _stage2(p) });
  } else {
    // 失敗對白（語言性暗示）
    DialogueModal.play(FAIL_LINES, { onComplete: () => _stage1Fail(p) });
  }
}
```

**玩家看不到 `15` 這個數字**。只看到結果對白。

### 失敗對白的內建暗示

每個階段失敗的對白**必須**包含「你是哪裡不夠」的明示：

- WIL 不夠 → 「要是能多等一下⋯⋯」「耐心沒夠」「心太急」
- AGI 不夠 → 「速度沒跟上」「慢了一步」「跑得不夠快」
- DEX 不夠 → 「手巧不夠」「沒抓穩」「手指沒那麼細」
- STR 不夠 → 「沒那個力道」「推不動」「扛不住」

玩家多試幾次，自然學會「要練哪項能過哪關」。

---

## 後續決策（完成後的選擇）

全通過後，**多一層選項**讓玩家決定結果：

例：抓老鼠完成後
- 放生 → moral/kindness +1，aff 梅拉 +5
- 殺掉 → moral/cruel +1，獎勵肉塊（食物 +20）
- 養 → 觸發小灰寵物線

每個選擇觸發不同 flag 與後續內容。

## 任務文件的標準格式

每個任務 md 檔（`docs/quests/<id>.md`）應包含以下章節：

```md
# 任務名
## 觸發條件（白天）
## 前置對白
## 門檻數值（程式內部用，不外顯）
## 階段 1：XX 判定
  ### 成功對白
  ### 失敗對白
## 階段 2：XX 判定
  ...
## 階段 3：XX 判定
  ...
## 後續決策
## 獎勵表
## 相關 flag
## 特性變化對白彙整
```

範本見 `docs/quests/mela-rat.md`。

## 與原本事件系統的差異

| 原本 | 新模式 |
|---|---|
| 單次 ChoiceModal | 連續 3-4 層 |
| 事件觸發即效果 | 事件觸發 → 接 → 等夜晚 → 判定 |
| 失敗就失敗，沒補償 | 失敗有分層慰問獎 |
| 一次性結果 | 階段推進 + 後續選擇 |

新模式更**有戲**、更**重視玩家投入**。
