## § 1 核心哲學

### 解決的問題
1. **裝備路徑單一**：目前只能等葛拉事件，玩家無主動權
2. **戰利品空虛**：競技場勝場只給金錢/名聲、零裝備
3. **強度膨脹失衡**：玩家強了之後對手不會強、後期割草
4. **沒有「刷裝」動機**：拿到的東西沒有縱向深度（只有 tier、沒有品質/詞綴）

### 三層設計
1. **品質**（縱向）：同 tier 同武器的數值倍率
2. **詞綴**（橫向）：特殊效果（吸血/燃燒等）
3. **多管道**（廣度）：主人賜 / 葛拉鋪 / 競技場掉 / 隔壁訓練所 / 黑鬍子

## § 2 品質系統（Quality）

### 5 級品質 + 顏色

| 品質 | 程式 ID | 顏色（HEX） | 數值修正 | 詞綴數 |
|---|---|---|---|---|
| 粗糙 | `crude` | `#888888`（灰） | -15% | 0 |
| 普通 | `common` | `#ffffff`（白） | ±0%（基準） | 0 |
| 精良 | `fine` | `#5a9aff`（藍） | +15% | 1 |
| 上等 | `superb` | `#c060ff`（紫） | +30% | 2 |
| 傳家 | `legendary` | `#ffaa20`（金） | +50% + 套裝特效 | 3 |

### 數值修正套用範圍
所有「攻擊面」數值：`ATK / ACC / CRT / CDMG / SPD / PEN`  
所有「防守面」數值：`DEF / blockChance / damageReduction`  
**不影響**：`hands / twoHanded / weaponClass / hitParts / special`（結構性欄位）

### 計算範例
```
hammer 基礎：ATK 16, ACC 5, CRT -2, CDMG 25, SPD -3, PEN 12
hammer (精良)：ATK 18, ACC 6, CRT -2, CDMG 29, SPD -3, PEN 14（+15%）
hammer (上等)：ATK 21, ACC 7, CRT -3, CDMG 33, SPD -4, PEN 16（+30%）
hammer (傳家)：ATK 24, ACC 8, CRT -3, CDMG 38, SPD -5, PEN 18（+50%） + 套裝特效
```

實作備註：負數值的 +/- 也按比例（CRT -2 ×1.15 = CRT -2.3 → round → -2 不變、上等才到 -3）

### 資料結構
```js
// src/content/weapons.js 武器物件擴充：
hammer: {
  id: 'hammer', name: '槌',
  // ... 原欄位 ...
  // 🆕 品質系統
  quality: 'common',         // crude/common/fine/superb/legendary（裝備上玩家身上才有；池子數據是 base）
  affixes: [],               // ['sharp', 'vampiric'] 詞綴 ID 陣列
  baseQuality: 'common',     // 用於回滾（強化失敗）
}
```

## § 3 詞綴系統（Affixes）

### 詞綴池（10 個）

| 詞綴 | 程式 ID | 效果 | 適用 |
|---|---|---|---|
| 鋒利的 | `sharp` | ATK +2 | 武器 |
| 精準的 | `precise` | ACC +3 | 武器 |
| 致命的 | `lethal` | CRT +5 | 武器 |
| 飛快的 | `swift` | SPD +2 | 武器 |
| 穿甲的 | `piercing` | PEN +3 | 武器 |
| 嗜血的 | `vampiric` | 命中時回 5% 損傷 HP | 武器 |
| 灼燒的 | `flaming` | 命中時對方 DoT 3 回合（每回合 5 損傷） | 武器 |
| 重擊的 | `crushing` | 暴擊 CDMG +20 | 武器 |
| 厚實的 | `sturdy` | DEF +3 | 護甲 |
| 反擊的 | `riposting` | 被擊後 30% 機率反彈 5 損傷 | 護甲 |

### 命名規則
- **武器名**：`{詞綴名}的 + {武器名}（{品質}）`
- 例：「鋒利的短劍（精藍）」「致命的嗜血的槌（上紫）」
- 多詞綴排序：按 ID 字母序（`lethal` 在 `vampiric` 前）

### 抽取規則
1. **掉落時抽**：依品質決定詞綴數、從可用池隨機抽
2. **不重複**：同件武器不會出現兩個 `sharp`
3. **裝備類型過濾**：護甲不會抽到 `vampiric`、武器不會抽到 `sturdy`
4. **抽到後鎖定**：詞綴是該件武器的永久屬性、不可重抽（除非升級品質）

### 升級品質時的詞綴處理
- 玩家在葛拉鋪強化品質：
  - 普白 → 精藍：**新增 1 個詞綴**（隨機）
  - 精藍 → 上紫：**新增 1 個詞綴**（共 2 個）
  - 上紫 → 傳金：**新增 1 個詞綴 + 套裝特效**（共 3 個 + 特效）
- 詞綴只增不換（保留舊的、加新的）

### 套裝特效（傳家專屬）

每件傳家裝備有獨特一句敘事 + 一個機械效果：

| 部位 | 名字 | 機械效果 |
|---|---|---|
| 武器（傳家） | 「葛拉之刃」/「葛拉之槌」（依 weaponClass） | 戰鬥開場 +5% ATK、命中時 1% 機率立刻致命一擊 |
| 胸甲（傳家） | 「葛拉之鎧」 | 受擊時 5% 完全格擋、HP < 30% 時 DEF +5 |
| 護臂（傳家） | 「主人傳家護臂」 | 連擊 +1（一回合多攻一次的小機率） |
| 護腿（傳家） | 「主人傳家護腿」 | 閃避率 +5% |
| 頭盔（傳家） | 「主人傳家頭盔」 | 對方暴擊機率 -10% |

## § 4 取得管道

### 4.1 主人賜（3 條護飾線、固定材質）

| 條線 | 材質 | 第 1 件（5 場勝） | 第 2 件（15 場勝） | 第 3 件（25 場勝） | 第 4 件 傳家（35 場 + 條件） |
|---|---|---|---|---|---|
| 護臂 | **布** | 布護臂（精藍） | 布精護臂（精藍 +1 詞綴） | 布上等護臂（上紫） | **三選一傳家** |
| 護腿 | **皮** | 皮護腿（精藍） | 皮精護腿（精藍 +1 詞綴） | 皮上等護腿（上紫） | **三選一傳家** |
| 頭盔 | **鐵** | 鐵頭盔（精藍） | 鐵精頭盔（精藍 +1 詞綴） | 鐵上等頭盔（上紫） | **三選一傳家** |

#### 主人賜起手就精藍（不給粗灰/普白）
理由：主人是「投資角鬥士」、不是慈善家。「打贏 5 場 → 值得藍裝」。

#### 第 4 件傳家三選一（身份定型）
35 場勝 + 5 連勝（A 評以上）+ 任一 S 評 → 觸發傳家事件：

```
[DialogueModal — 主人召見]

主人：⋯⋯到了這份上、給你選一件吧。
（他指著三個盒子。）

選項：
  ▸ 布傳家斗篷（DEX +5、SPD +3、AGI +2）— 風 build
  ▸ 皮傳家護甲套（全屬性 +3）— 平衡 build
  ▸ 鐵傳家鎧甲（DEF +8、CON +5）— 鐵壁 build

主人（選後）：拿去。
主人：我問了葛拉。剩下兩件、他熔掉給其他兄弟。
```

**選了不能換**——身份宣告。剩下兩件熔給其他兄弟（呼應葛拉信用點 / 新血池）。

#### 對白配合（每階段）
- 5 場：「⋯⋯不錯。給你件像樣的。」
- 15 場：「我看你能再走遠一點。」
- 25 場：「我認真考慮投資你了。」
- 35 場+：「⋯⋯選一件吧。我家族的東西。」

#### 主人不給胸甲、武器（葛拉專屬）
胸甲是葛拉故事核心、武器是葛拉個人任務。主人線只給護飾。

### 4.2 葛拉鐵匠鋪（玩家主動 UI）

#### 美術資產
- **背景圖**：`asset/image/blacksmith.png`（鐵匠鋪內景、火爐 + 鐵砧 + 暗色調）
  - 所有葛拉對話場景共用：進坊、UI、鋪內 ChoiceModal、主武器之路各階段獨白
  - 實作建議：寫一個 `Stage.playForgeBackdrop()` 共用元件、餵這張圖當背景
- **角色顯示**：**沒大頭照、用名字**（`{ speaker: '葛拉', text: '...' }`）— 跟全遊戲既有 DialogueModal 一致
- **葛拉開場首句範本**：「今天來幹啥？」（隨好感變化、見階段對白）

#### 解鎖事件：塔倫來交代

**觸發條件**：主人好感 ≥ 50 **或** 累計勝場 ≥ 10 **或** 首次 S 評（哪個先到）

**過場**（DialogueModal）：
```
塔倫：主人讓我跟你說一件事。
塔倫：⋯⋯你接下來訓練時間能去找葛拉。
塔倫：升武器、強化、修，你要什麼跟他說。
塔倫：主人付一半，剩下你自己出。
塔倫：⋯⋯主人想多看幾場像上次那種的。
（他轉身要走，又停下。）
塔倫：葛拉那老頭挑剔，別惹毛他。
```

**解鎖**：
- 訓練場行動列出現「**去找葛拉**」鈕（消耗 1 個 slot）
- 設 flag `gra_forge_unlocked`
- 鋪內所有費用 ×0.5（「主人付一半」永久）

#### 鐵匠鋪 UI 設計

```
┌─ 葛拉的鐵匠鋪 ──────────────────────────────────┐
│                                                  │
│  葛拉：「想動哪把？」                            │
│                                                  │
│  [選一件武器/護甲]                               │
│   ▸ 短劍（精藍）·  詞綴：鋒利的                  │
│   ▸ 槌（普白）·  無詞綴                          │
│   ▸ 匕首（粗灰）·  無詞綴                        │
│   ▸ 皮甲（普白）·  無詞綴                        │
│                                                  │
│  選擇操作：                                      │
│   [強化品質]   25 金（葛拉信用 -10 可抵 50%）     │
│   [升 Tier]    100 金 + 葛拉好感 60               │
│   [鍛新詞綴]   50 金 + 上紫以上                   │
│   [離開]                                         │
│                                                  │
│  葛拉信用：  37 點                               │
└──────────────────────────────────────────────────┘
```

#### 三大功能

| 功能 | 條件 | 費用（主人付半後） | 效果 |
|---|---|---|---|
| **強化品質** | 該件當前品質 < 上紫 | 12 金 + 信用 5（可選） | 品質 +1 階、自動加 1 詞綴 |
| **升 Tier** | 該件當前 tier < 3 + 葛拉好感 60 | 50 金 | Tier +1（T1→T2→T3） |
| **鍛新詞綴** | 上紫以上 + 葛拉好感 70 | 25 金 + 信用 10 | 多 1 個詞綴（最多 3 個） |

#### 強化失敗機率（給戲劇感）
- 普白 → 精藍：**0% 失敗**
- 精藍 → 上紫：**10% 失敗**（武器降回普白、葛拉「⋯⋯抱歉」）
- 上紫 → 傳金：**禁止**（傳金只能事件取得、不能強化）

**保險選項**：付 +20 金「保險費」→ 0% 失敗

#### 冷卻
- 每次操作 → **3 天冷卻**（葛拉花時間）
- 玩家可規劃資源、但不能無限刷強化

### 4.3 競技場戰利品（隨機刷感）

#### 觸發機率（依評分）

| 評分 | 觸發掉落機率 | 觸發後品質分布 |
|---|---|---|
| **S** | 80% | 20% 紫 / 30% 藍 / 50% 綠 |
| **A** | 60% | 20% 藍 / 30% 綠 / 50% 白 |
| **B** | 40% | 20% 綠 / 30% 白 / 50% 灰 |
| C/D | 0% | — |

戰鬥狂熱中：**所有觸發機率 +10%**（S→90%、A→70%、B→50%）

#### 戰利品內容（規則 A：對手掉啥拿啥）
- **武器**：對手主手武器 → 玩家拿到（含詞綴）
- **護甲**：對手胸甲 → 25% 機率掉
- **護飾**：對手有的話 → 15% 機率掉
- **副手**：對手副手（盾/匕首）→ 30% 機率掉

**詞綴保留規則**：對手身上是「鋒利的短劍」→ 玩家拿到「鋒利的短劍」（保留全詞綴）

#### 配套：對手強度拉強

| 對手 | 屬性 vs 玩家當期 | 裝備品質池 | 詞綴 |
|---|---|---|---|
| **rookie** | -2~-3 | 粗灰 80% / 普白 20% | 0 |
| **gladB** | **持平** | 普白 50% / 精藍 40% / 上紫 10% | 0-1 |
| **vet** | **+1~+3** | 精藍 50% / 上紫 40% / 傳金 10% | 1-2 |
| **champion** | **+5~+8** | 上紫 60% / 傳金 40% | 2-3 |

「**想拿 S 評就要配裝刷屬性才配得上**」—— 對手強度逼玩家配裝。

#### 隨玩家進度動態升級對手池
- 玩家 fame 達門檻：對手池整體往上一階
- rookie 出現率逐漸下降、vet/champion 出現率上升
- 公式：`rookieRate = max(0.1, 0.8 - fame/200)` 等（細節待調）

## § 5 葛拉信用點系統

### 設計目的
- 多餘裝備有處置路徑（不變垃圾）
- 葛拉的「資源」有合理來源
- 連到 NPC 故事感（「我熔給其他兄弟用」 → 補新血池武器）

### 兌換表

| 上交品質 | 葛拉信用點 |
|---|---|
| 粗灰 | 1 |
| 普白 | 2 |
| 精藍 | 4 |
| 上紫 | 8 |
| 傳金 | **不可上交**（你的命） |

### 用途

| 用途 | 信用消耗 | 效果 |
|---|---|---|
| 抵強化費 50% | 5-10 點 | 對應金錢費用減半 |
| 兌換稀有材料 | 30 點 | 獲得「鋼錠」（用於升 T3） |
| 解鎖特殊強化 | 50 點 | 多一次強化機會（突破上紫上限） |
| 兌換雙刃藍圖等秘法 | 100 點 | 接現有藍圖機制 |

### 上交對白
```
（你把舊裝備放在葛拉的鐵砧上。）

葛拉：給我吧。我熔一熔給其他兄弟用。
葛拉：⋯⋯記下了。{N} 點。
```

### 跟新血池的連動
- 新血池武器補充來源 = 玩家上交的裝備
- 上交 5 件後、訓練場新血武器會出現「葛拉鍛的版本」（純風味）
- 玩家在訓練場看到「自己以前的劍」掛在新人身上

```
裝備庫存滿了
你拿到「致命的短劍（上紫）」、但儲物已滿。

選一件處置：
  ▸ 上交葛拉（換 8 信用點）
```

## § 7 對手強度重新校準（呼應 § 4.3）

### 設計鐵則
1. **對手永遠跟玩家差半階**（玩家精藍時對手普白、玩家上紫時對手精藍）
2. **rookie 給輾壓快感、vet 給緊張、champion 給絕望**
3. **主敘事 boss 永遠頂規**（無視玩家裝備、固定上紫/傳金 + 3 詞綴）

### Boss 戰設計鐵則

「**沒認真配裝刷屬性的玩家別奢望過**」—— 玩家原話。

| 設計點 | 規格 |
|---|---|
| 裝備 | 上紫 + 2-3 詞綴（傳家也可能）|
| HP | × 1.5（boss 加成）|
| ATK | × 1.3 |
| 特殊招式 | 1-3 個（單回合大爆發 / 召喚援軍 / debuff 玩家）|
| 預期玩家 | T3 上紫 + 護飾傳家齊 + 主屬性 35+ |
| 裸玩家通關率 | < 10% |

### 三大 boss 候選

| boss | 出現時機 | 性格 | 設計重點 |

## § 11 風險與取捨

### 風險 1：詞綴讓戰鬥計算複雜化
**緩解**：詞綴效果都是簡單疊加（+ATK 2 等），不影響戰鬥引擎核心邏輯

### 風險 2：玩家裝備膨脹後對手跟不上
**緩解**：對手吃同一套品質+詞綴系統、動態升級對手池

### 風險 3：葛拉鋪 UI 工作量大
**緩解**：複用現有 ChoiceModal + 簡單 list + 一頁式設計、不做複雜分頁

### 風險 4：儲物上限導致玩家煩躁
**緩解**：上交葛拉換信用點是無痛處置（還賺）、滿庫存提示 ChoiceModal 直觀

### 取捨：放棄了什麼
- ❌ 武器磨損機制（user 否決、麻煩沒意義）
- ❌ Diablo 式「20+ 詞綴池」（簡化到 10 個）
- ❌ 詞綴隨機重抽（一次定型、保留風味）
- ❌ 主人賜可選材質（改固定材質、玩家湊齊靠別處）
