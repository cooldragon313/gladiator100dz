## 🩸 起手傷勢系統（2026-04-19 重整）

**全部傷統一到 wounds 系統**（不再用 ailments 給起手）：

- 5 部位（head / torso / arms / legs / **mind**）
- 12 種一般傷（4 部位 × 3 級）+ 4 種特殊傷（concussion / achilles_tear / insomnia / depression）
- 每種獨立 3% 擲骰 → 期望 ~0.48 / 玩家
- ~39% 玩家起手有某種傷，~9% 有 2+ 個傷

詳細見 `docs/CODEX.md § 傷勢字典`。

`player.ailments` 保留給 Phase 2 的**非傷狀態**（中毒 / 流血 / 瘟疫 / 病痛）— 起手預設**空**。

#### ⚠️ Day 1 主動隱瞞
**奧蘭 Day 1 自我介紹時只說「磨坊裡來的」，刻意不提主人家 6 年**：
- **原因**：怕玩家知道他跟主人有舊緣而不信任（誤以為他是主人的眼線）
- **策略**：先以「跟你一樣的受害平民」建立同盟
- **揭露時機**：Day 40+（aff ≥ 40）storyReveal 才透露傭人身分
- **深度揭露**：Day 70+ 才完整說出偷錢 / Livia 求情的真相

這意味：
- 「磨坊裡來的」**不是謊言，是簡化版實話**（他**出身**磨坊是真的）
- 他對玩家的**善意是真的**，但**信任需要時間建立**
- Day 40+ 揭露時是關鍵情感節點（玩家可能覺得被騙 / 也可能理解）

## 使用規則

### 設計新對白 / 事件時

1. **第一步：查 CANON**
   - 角色的年齡、時間線符合嗎？
   - 角色知道這件事嗎？（參考知識矩陣）
   - 地理上說得通嗎？

2. **第二步：發現未決事實 → 問使用者**
   - 不 extrapolate「訓練場 2 年」「認識 5 年」這種沒講過的數字
   - 用「大約」或留白，等使用者確認

3. **第三步：寫完同步 CANON**
   - 新角色年齡 / 新時間點 / 新知識流動 → 寫進對應章節
   - commit 時一併更新

### 衝突處理

- **CANON 與既有程式碼衝突** → 優先修程式碼對齊 CANON（CANON 是源頭）
- **CANON 與其他 md 衝突** → 合併到 CANON，其他改為引用
- **CANON 與使用者新設定衝突** → 使用者當場決定，當場改 CANON
