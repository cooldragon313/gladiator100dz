## § 0 設計哲學

### 0.1 核心
> **「先吃苦、後甜頭、最後感動。」** 玩家用前 30 天承擔代價、中期投資、後期得到一個真正屬於自己的兄弟。

### 0.2 跟殺索爾的對比

| 面向 | 殺索爾 | 救索爾 |
|---|---|---|
| 立即回饋 | 木牌掛件 LUK +1、主人 +5 | 主人 -15、長官 -10、**200 銅幣債務**（買價 100 + 罰金 100）|
| 中期路線 | 競技場日常 | 索爾養傷 → 默默回報 → 治療抉擇 → 義肢投資 |
| 後期回報 | LUK 永久加成 | 戰友、戰利品分享、結局忠誠 |
| 主題 | 「殘酷世界」 | 「義氣與重生」 |

→ 不是「比較好的選擇」、是**完全不同的遊戲體驗**。

### 0.3 邏輯一致性原則

- 競技場**一直開**（不會因玩家選擇關閉）
- 但**新人都要訓練 30 次才能上**（共用門檻、不分救不救）
- 救索爾的差異是**經濟代價**（200 銅幣 + 私財凍結）、不是內容鎖
- 索爾的傷由**主人付錢給老默治**（主人捨不得讓 100 銅幣的投資白費）→ 玩家不用額外付醫藥費

### 1.7 私財扣抵機制

```js
// 任何 modMoney(+) 都要先走這個 wrapper
function _modMoneyWithSkim(delta) {
  if (delta > 0 && Flags.has('master_skim_active')) {
    const debt = Flags.get('debt_to_master', 0);
    if (debt > 0) {
      const taken = Math.min(delta, debt);
      Flags.set('debt_to_master', debt - taken);
      delta -= taken;
      addLog(`✦ 主人扣下 ${taken} 銅幣（剩 ${debt - taken} 還清）`, '#aa6666', true);
      if (debt - taken <= 0) {
        Flags.unset('master_skim_active');
        Flags.set('master_debt_cleared', true);
        _triggerDebtClearedCeremony();   // § 1.8 還清演出
      }
    }
  }
  Stats.modMoney(delta);
}
```

**扣抵範圍**：
- ✅ 跑腿事件給的錢
- ✅ 競技場勝場獎金
- ✅ 戰利品變賣（後期 NPC 反應給的小費也算）
- ❌ 不扣老默的免費治療（那不是錢）
- ❌ 不扣事件給的物品（書、藥水、武器等）

### 1.7b 🆕 私財凍結期老默賒帳（2026-05-13）

**問題**：私財凍結 = 玩家錢包 0 + 收入全扣抵。受傷需要治療、玩家付不起。

**邏輯**：
- 老默看到玩家沒錢、不會放著等死（主人不准）
- 主人替付治療費、但**加倍記到 debt**（cold_investor 敲詐）

**機制**：
```js
// 當玩家錢不夠、且 master_skim_active：
if (playerMoney < cost && Flags.has('master_skim_active')) {
  debtBilled = cost * 2;                           // 加倍
  Flags.set('debt_to_master', oldDebt + debtBilled);
  // 不從玩家錢包扣（沒錢可扣）
  // 治療正常進行
}
```

**對白**：
```
老默：「⋯⋯沒錢？沒關係。」
老默：「主人讓我看著你、不是讓你死。」
老默：「賬記他頭上。」
老默：「⋯⋯不過你欠的、又多了 {cost × 2}。」
```

**設計意涵**：
- 玩家受傷不會被卡死（治療永遠進行）
- 但 debt 雪球累積 → 玩家更想避免受傷
- 主人 cold_investor 邏輯一致（投資 vs 敲詐）

### 2.2 偷懶遮掩（取代擋鞭）

⚠️ 索爾那天 mood -5（替罪、緊張）

**條件**：
- `Flags.has('sol_grateful_oath')`
- 索爾仍跛（`!Flags.has('sol_prosthetic_done')`）
- 玩家當天動作 = 偷懶 / 擺爛

**程式 hook**：在 `actions.js` 或 `main.js` 的偷懶結算處插入：
```js
if (Flags.has('sol_grateful_oath') && !Flags.has('sol_prosthetic_done')
    && _solPresent() && Math.random() < 0.30) {
  SolArcEvents.tryCoverSlacking();
}
```
