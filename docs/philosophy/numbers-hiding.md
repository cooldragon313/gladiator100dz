# 數字隱藏原則

> 這份哲學適用於：好感度、屬性門檻、任務需求、熟悉度、道德軸、EXP 以外的所有「玩家決策相關的數值」。
> 最後更新：2026-04-18

---

## 核心原則

**玩家不該看到赤裸的數字。**
所有機制必須透過**對話、顏色、敘述、後果**呈現。

| 錯誤 | 正確 |
|---|---|
| ❌ 「熟悉度 40/40」 | ✅ NPC 對你說「跟你練起來不一樣了」 |
| ❌ 「需求 WIL 15」 | ✅ 梅拉：「這小子狡猾，你要夠耐心」 |
| ❌ 「好感 -10」 | ✅ NPC 名字變灰、對話句型轉冷 |
| ❌ 「道德軸 reliability +2」 | ✅ 晨思「我好像成了某種人」 |

---

## 例外情況（可以秀數字）

以下數字**是允許的**，因為它們是「資源管理」而非「角色內在狀態」：

- **EXP**（玩家知道自己累積了多少，才能理性決定花在哪）
- **體力 / HP / 飽食 / 心情 / 名聲 / 金錢**（資源條，需要具體數值決定行動）
- **一日時段數**（4 slot，玩家要規劃時間）
- **日數**（百日倒數是遊戲的核心 stakes）

這些是**外在資源**，玩家跟這些數字互動時不會破壞角色沉浸感。

## 任務需求的標準表達

### 寫錯
```
📋 抓老鼠
  需求：WIL 15 · AGI 18 · DEX 18
  [接任務]
```

### 寫對
```
📋 梅拉塞的請託
  「這小子狡猾得很——你要有足夠的耐心，
    等得住。速度要夠快——他跑起來像一縷煙。
    手要巧——他滑溜得像油。」
  「……三樣缺一不可。你看看自己夠不夠。」
  [我試試] [下次吧]
```

**玩家必須從語境推論**「耐心=WIL、速度=AGI、手巧=DEX」。但數字完全不秀。

## 失敗回饋的標準做法

**失敗對白要告訴玩家「你是哪裡不夠」**，但仍不秀數字。

### 例：抓老鼠三階段判定
| 階段 | 檢測 | 失敗對白 |
|---|---|---|
| 1. 等待 | WIL | 「早知道該耐心點等他放下戒心……」 |
| 2. 追趕 | AGI | 「這賊小子跑那麼快——看來沒點速度會追不到」 |
| 3. 抓取 | DEX | 「指尖就差那麼一點——手要更巧才抓得住這東西」 |

玩家失敗幾次後會學會：**「哦，我要練 XX 才能過這關」** — 但他心裡說的是「速度」，不是「AGI」。這個翻譯由玩家自己完成。

## 顏色的角色

**顏色** = 數字的替身。玩家看不到「好感 60」，但能看到：
- 名字**金色** → 盟友級
- 名字**紅色** → 宿敵級
- 卡片**邊框發光** → 剛解鎖新狀態
- 屬性發光**紫光** → 高階協力（×1.8）

顏色是唯一的「量化視覺暗示」。文字永遠是定性描述。

## 🆕 對話框 vs 日誌：獎勵訊息的分工（D.28）

### 核心規則

| 位置 | 秀獎勵訊息？ | 範圍 |
|---|---|---|
| **DialogueModal（對話框）** | ❌ 不秀 | 純敘事 / 情感 / 對白 |
| **ChoiceModal（選擇框）** | ❌ 不秀 | 選項本身不含 `+5 HP` 這類 |
| **addLog（日誌區）** | ✅ 秀 | 所有獎勵/懲罰都該反映 |

### 為什麼

- 對話框在情感/沉浸時刻出現，秀數字會**打斷戲**
- 但玩家仍需要知道**我選了這個得到/失去了什麼**
- 日誌是**事後總結**，不影響當下情境，顯示獎勵不會破壞沉浸

### 格式規範（日誌內）

| 類型 | 秀法 | 範例 |
|---|---|---|
| HP / 體力 / 飽食 / 心情 | emoji + 數字 | `❤️-5` / `⚡+15` / `🍖-10` / `💭-3` |
| 名聲 | 中文 + 數字 | `名聲+5` / `名聲-3` |
| 金錢 | 💰 + 數字 | `💰+20` / `💰-15` |
| EXP / 屬性 | KEY + 數字 | `STR EXP+8` |
| 好感 | NPC 名 + 箭頭**（不秀數字）** | `達吉 好感↑` / `塔倫長官 好感↓` |
| 道德軸 | **不秀**（只在獲得/失去特性時另外 log） | — |
| 旗標 | **不秀**（除非是重大劇情旗標） | — |

### 實作方式

- **`Effects.apply()` 自動產生獎勵總結 log**（D.28 新增）
  - 每個 effect 套用時會 push 到 `ctx._summary`
  - 所有 effect 跑完後一起 `addLog('（...）')`
  - 格式：`　（❤️-5 · 達吉 好感↑ · 奧蘭 好感↑）`
- **直接呼叫 `modAffection` / `modVital` 的地方** → 改用 `Effects.apply()` 或手動 addLog
- **不想自動 log 的情境** → 傳 `{ silent: true }` 給 `Effects.apply()`

### 範例

**選擇後的完整日誌輸出**：
```
[resultLog]     你走過去，撿起碗，放回奧蘭面前。赫克托看著你——那眼神你會記住。
[auto-summary] 　（赫克托 好感↓ · 奧蘭 好感↑ · 卡西烏斯 好感↑ · 梅拉塞 好感↑）
```

玩家看到：**敘事 + 獎勵**。兩行互補。

### 這不違反數字隱藏原則

- 體力/HP/飽食/心情 = **資源**，一直以來就秀數字（HUD bar 也秀）
- 好感 = 只秀**方向**（↑↓），不秀數字
- 道德軸 = **完全隱藏**（只在特性改變時才暴露）

所以**沒有破戒**，只是把原本藏在暗處的變動**明確告知**玩家。

## 設計文件內部 vs 遊戲內

**這條線很重要**：

| 文件類型 | 可秀數字 | 例 |
|---|---|---|
| 程式碼 | ✅ | `if (aff >= 30) tier = 1;` |
| 設計 md（`docs/` 下） | ✅ | 「WIL ≥ 15 判定通過」 |
| CHANGELOG | ✅ | 「SLEEP_STAMINA_GAIN 40 → 45」 |
| **遊戲內 UI / 對話 / Modal** | ❌ | 絕對禁止 |
| **玩家看得到的任何文字** | ❌ | 絕對禁止 |

---

## 如果要破戒（暫不考慮）

未來如果真的要給硬核玩家一個「看數字」模式，做法會是：
- 設定頁加一個「開發者模式」toggle
- 預設關閉，只有開了才顯示
- **不要**當作一般功能

目前為止：**全遊戲零數字**。

---

## 對 AI 開發者的提醒

寫任何新功能前檢查：
1. UI 上會出現數字嗎？→ 改成條、顏色、文字描述
2. Modal 裡有數字嗎？→ 改成 NPC 對話的口語描述
3. log 裡有數字嗎？→ 通常可以（資源變化），但故事類 log 要避免
4. 失敗回饋有數字嗎？→ 絕對不行，改成角色口氣的反省對白

**一個句子的測試**：把這段話念給朋友聽，他會覺得你在玩遊戲還是讀說明書？
