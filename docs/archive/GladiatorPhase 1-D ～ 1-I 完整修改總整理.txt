Phase 1-E：飢餓 / 疲勞 / 心情閾值事件
涉及檔案： main.js（doAction 內）

強制中斷（訓練前 pre-check，100% 機率）
體力 ≤ 10（力竭）：取消訓練，消耗時段，stamina +5 作為強制休息
心情 ≤ 9（崩潰）：取消訓練，消耗時段，mood −5（情緒螺旋加深）
效率軟修正（訓練中，可疊加）
心情 ≥ 80（40% 機率）：✨ 心流 → 訓練效率 ×1.5
體力 ≤ 25（25% 機率）：擺爛 → 訓練效率 ×0.4
飽食 ≤ 30（25% 機率）：輕度飢餓 → 訓練效率 ×0.75
三個修正可同時發生，最終乘進 finalSynergyMult = synergyMult × thresholdMult。Log 尾端顯示如「（心流 ×1.5）（飢餓 ×0.75）」。

受傷系統 v2（Phase 1-E 同步重構）
涉及檔案： main.js（doAction 受傷判定區塊）

舊邏輯問題：stamina 低 → 受傷率高，但 Phase 1-E 同時說 stamina 低 → 擺爛（矛盾）。

新邏輯：訓練強度決定受傷，不是體力高低

條件	受傷加成
超負荷比（消耗/當前體力）≥ 2.0	+50%
超負荷比 ≥ 1.5	+35%
超負荷比 ≥ 1.0（剛好耗盡）	+20%
超負荷比 ≥ 0.7	+8%
每點絕對透支量	+1.5%
協力 ×6.0 以上	+30%
協力 ×3.0 以上	+18%
體力 ≤ 20（輕微加成）	+5%
擺爛（thresholdMult < 1）→ 完全跳過受傷判定
超負荷比 ≥ 1.5 或協力 ≥ ×3 → 重傷（−25 stamina，−20 mood）
其他 → 輕傷（−12 stamina，−8 mood）
