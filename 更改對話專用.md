# 更改對話專用 · 對白編輯入口

> **這是改遊戲對白唯一要看的檔案。**
> 找到事件 → 點連結到 .js 檔 → 改 `text:` 字串 → 存檔 → 重整瀏覽器。
>
> 🚨 **規則**：以後所有新事件、新對白、新 NPC 出場，**都要在本檔對應區塊加一行連結**。
> 不然下次想改對白時會找不到。

---

## 📝 對白格式速記（90% 場景）

```js
const lines = [
  { text: '（旁白用括號、心理描寫）' },                          // 旁白
  { speaker: '梅拉', text: '⋯⋯孩子。', color: '#9dbf80' },     // NPC 講話
  { text: '⋯⋯', color: '#666' },                              // 沉默 / 過場
  { text: '（——重點）', color: '#ff8866' },                    // 暖紅強調
  { text: '（猛然震一下）', effect: 'shake' },                  // 震動效果
];
DialogueModal.play(lines, { onComplete: () => { /* 結束 callback */ } });
```

**改步驟（4 步走）**：
1. 在下方表格找到事件、點連結
2. .js 檔打開後 Ctrl+F 搜函式名（例 `_playFarmboyRecognition`）
3. 找到 `lines = [` 陣列、直接改 `text:` 字串
4. 存檔 → 重整瀏覽器、立刻生效（不需要 build）

## 🚨 改對白前的注意事項

1. **NPC 必須在場才能講話** — 用 `_isPresent('npcId')` 守衛、不然會出現「死人講話」bug
2. **`Game.addLog` not bare `addLog`** — 外部模組（quests/npc）要用 `Game.addLog`、不能 bare（CLAUDE.md 第 12 條鐵律）
3. **每個 ChoiceModal 選項都要有反饋** — 對白 + 視覺特效（震動/閃光/音效）至少一項（CLAUDE.md 第 11 條鐵律）
4. **內心 OS 風格** — 括號內心描寫不用「你」開頭、省略主詞或用「我」（memory `feedback_monologue_style.md`）
5. **改完對白要驗收** — 用 `testJump('dayNN')` + `Game.testChoice` 等 debug 工具驗
